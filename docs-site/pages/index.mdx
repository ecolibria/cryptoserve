## How It Works

<Steps>
### Create an Identity

Sign in to the dashboard and create an identity for your application.

### Download Your SDK

Get a personalized SDK with your identity embedded - no configuration required.

```bash copy
pip install https://your-server/sdk/download/YOUR_TOKEN/python
```

### Start Encrypting

Use the SDK with a single line of code. The right algorithm is selected automatically.

```python copy
from cryptoserve import CryptoServe

crypto = CryptoServe(app_name="my-app", team="platform")

# Encrypt with automatic algorithm selection
encrypted = crypto.encrypt_string("user@example.com", context="user-pii")

# Decrypt seamlessly
email = crypto.decrypt_string(encrypted, context="user-pii")
```
</Steps>

## 5-Layer Context Model

CryptoServe automatically selects the optimal algorithm based on your data's requirements:

| Layer | Purpose | Example |
|-------|---------|---------|
| **Data Identity** | What type of data? | PII, PHI, PCI, credentials |
| **Regulatory** | Compliance requirements | HIPAA, PCI-DSS, GDPR |
| **Threat Model** | Who are adversaries? | Nation-state, organized crime |
| **Access Patterns** | How is data accessed? | High-frequency, batch, rare |
| **Technical** | Infrastructure constraints | FIPS mode, latency limits |
